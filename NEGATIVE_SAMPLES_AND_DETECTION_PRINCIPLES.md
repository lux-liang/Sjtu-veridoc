# 负样本构造与检测原理说明

## 1. 目的

当前数据集中虚假样本太少，尤其是 PDF 虚假样本只有 3 个，无法支撑可靠训练。

下一阶段需要按材料类型和伪造方式系统构造负样本，并明确每类负样本对应的检测原理。

本文档用于说明：

- 每类材料应该补充哪些负样本。
- 每类伪造可以通过哪些特征检测。
- 当前系统能检测什么。
- 后续更强方案可以如何实现。

## 2. 负样本设计原则

负样本不是简单把文件放进 `fake` 文件夹，而是要覆盖真实业务中常见的伪造方式。

建议每个材料类型至少构造以下几类负样本：

| 伪造方式 | 说明 |
| --- | --- |
| 文字替换 | 修改姓名、身份证、公司名、日期、金额、合同编号 |
| 数字篡改 | 修改金额、余额、税额、贷款金额、授信额度 |
| 印章贴图 | 把印章截图贴到 PDF 或图片上 |
| 签名贴图 | 把签名图片贴到合同、回单、证明文件上 |
| 局部遮盖重写 | 用白块/背景块覆盖原内容后重新写字 |
| 拼接合成 | 多个页面、多个来源文件拼成一份材料 |
| 截图转 PDF | 用截图或扫描图重新生成 PDF |
| 元数据伪装 | 修改 Creator、Producer、PDF 版本、生成工具痕迹 |
| 格式转换伪造 | Word/WPS/图片/PDF 多次转换导致结构异常 |
| 模板伪造 | 套用真实材料模板，替换关键字段 |

## 3. 按材料类型补充负样本

### 3.1 征信报告 credit_report

建议负样本：

| 样本类型 | 具体修改 |
| --- | --- |
| 身份信息篡改 | 修改姓名、证件号、手机号 |
| 贷款信息篡改 | 修改贷款余额、逾期次数、账户状态 |
| 查询记录篡改 | 删除或修改查询机构、查询日期 |
| 页眉页脚拼接 | 复制真实报告页眉页脚，拼接虚假正文 |
| 截图重组 | 把页面截图后局部改字，再转成 PDF |
| 局部覆盖 | 用白色矩形覆盖逾期记录，再补文字 |

可检测线索：

- 局部覆盖会产生小图层、SMask、透明蒙版。
- 截图重组会导致整页大图、多图片对象。
- 字体替换会导致字体对象不一致、字体警告。
- 页面尺寸或 Producer 与正常征信报告不一致。
- 文本层与图像层不一致。

### 3.2 合同 contract

建议负样本：

| 样本类型 | 具体修改 |
| --- | --- |
| 金额篡改 | 修改合同金额、付款比例、违约金 |
| 日期篡改 | 修改签署日期、生效日期、到期日期 |
| 主体篡改 | 修改甲乙方名称、统一社会信用代码 |
| 印章贴图 | 将其他合同印章截图贴入当前合同 |
| 签名贴图 | 将签名图片贴到签署栏 |
| 页替换 | 替换合同某一页或插入新页 |

可检测线索：

- 印章或签名贴图常有透明通道、边缘锯齿、分辨率不一致。
- 局部文字替换会导致字体、颜色、基线和字符间距异常。
- 页替换可能导致页尺寸、Producer、字体集合不一致。
- 合同金额与大写金额、付款条款可能语义不一致。

### 3.3 发票 invoice

建议负样本：

| 样本类型 | 具体修改 |
| --- | --- |
| 金额篡改 | 修改价税合计、税额、不含税金额 |
| 抬头篡改 | 修改购买方名称、纳税人识别号 |
| 项目篡改 | 修改货物或服务名称、数量、单价 |
| 二维码替换 | 替换或模糊发票二维码 |
| 印章贴图 | 叠加发票专用章图片 |
| 截图伪造 | 用图片编辑工具改发票截图 |

可检测线索：

- 发票金额字段之间存在强约束，可做规则校验。
- 二维码区域质量、结构、位置可检测。
- 印章贴图有边缘、透明、颜色分布异常。
- 图片发票可做 PS 痕迹检测和 OCR 一致性校验。

### 3.4 银行流水 / 网银页面 bank_page

建议负样本：

| 样本类型 | 具体修改 |
| --- | --- |
| 余额篡改 | 修改账户余额、可用余额 |
| 流水篡改 | 修改交易金额、交易对手、摘要 |
| 删除交易 | 遮盖或删除不利交易记录 |
| 页面拼接 | 不同账户或不同日期截图拼接 |
| 截图转 PDF | 浏览器截图后修改，再转 PDF |

可检测线索：

- 表格行距、字体、对齐方式可能不一致。
- 数字区域压缩噪声与背景不一致。
- 截图转 PDF 通常整页大图，缺少正常文本层。
- 余额与流水加减关系可做逻辑校验。

### 3.5 回单 / 结算单 receipt / settlement_statement

建议负样本：

| 样本类型 | 具体修改 |
| --- | --- |
| 金额篡改 | 修改付款金额、收款金额 |
| 户名篡改 | 修改收付款账户名 |
| 账号篡改 | 修改银行卡号或对公账号 |
| 银行章贴图 | 贴入电子章或扫描章 |
| 流水号篡改 | 修改业务编号、交易流水号 |

可检测线索：

- 电子章贴图存在透明、边缘、颜色和分辨率异常。
- 账号、户名、金额、时间字段有格式和逻辑规则。
- 回单模板高度结构化，适合做模板对齐和版式检测。

## 4. 五类核心风险检测原理

### 4.1 PDF 对象结构异常

当前已实现一部分。

检测对象：

- `/SMask`
- `/Subtype /Image`
- `pdfimages -list`
- `pdfinfo`
- 图片对象数量
- 大图、小图层数量
- 字体警告

原理：

正常业务系统导出的 PDF 通常对象结构比较稳定；伪造 PDF 往往经历截图、贴图、覆盖、二次导出，容易产生额外图片层、透明蒙版和字体异常。

当前规则示例：

| 检测项 | 风险原因 |
| --- | --- |
| `smask_count >= 1` | 可能存在透明贴图或局部覆盖 |
| `font_warning_count >= 1` | 字体嵌入或 PDF 结构异常 |
| `large_image_count >= 1 && small_overlay_count >= 3` | 整页扫描图上叠加局部图层 |

### 4.2 图片 PS 痕迹检测

当前已实现基础版，入口为 `src/analyze_visual_forensics.py`。目前主要分析 PDF 渲染后的首页或输入图片。

已接入技术：

| 方法 | 原理 |
| --- | --- |
| ELA Error Level Analysis | 多次 JPEG 压缩后，篡改区域误差分布通常不同 |
| 噪声一致性分析 | 同一相机/扫描区域噪声应相对一致，贴图区域噪声不同 |
| 边缘与锐度检测 | 粘贴区域边缘可能过锐、过糊或与背景不一致 |

当前特征字段：

| 字段 | 说明 |
| --- | --- |
| `ela_score` | 重压缩误差均值，局部二次压缩会抬高 |
| `block_variance_score` | 64x64 分块噪声差异，贴图区域噪声可能突变 |
| `edge_density` | 页面边缘密度，局部覆盖/粘贴会产生硬边界 |
| `red_stamp_score` | 红色像素比例，用于红章/红色标记初筛 |

当前风险原因：

- `high_ela_recompression_error`
- `local_noise_block_inconsistency`
- `dense_edge_or_paste_boundary`
- `red_stamp_like_region`

适用场景：

- 发票截图改字
- 网银截图改金额
- 扫描件局部 PS
- 印章或签名贴图

仍需增强：

- 从首页扩展到前 N 页或风险页抽样。
- 增加 JPEG DCT 频域块伪影。
- 对扫描件建立来源分层阈值，避免正常扫描件被噪声/边缘误报。
- 增加局部热区定位，而不是只输出整页分数。

### 4.3 印章贴图检测

当前已实现基础版并接入综合评分，入口为 `src/analyze_visual_forensics.py` 和 `src/extract_pdf_features.py`。

已实现特征：

| 特征 | 说明 |
| --- | --- |
| `smask_count` | PDF 透明蒙版，PNG 印章贴图常见 |
| `red_stamp_score` | 红色像素占比，判断是否存在红章区域 |
| `red_component_count` | 红色连通组件数量 |
| `max_red_component_ratio` | 最大红色组件面积占比 |
| `red_component_edge_contrast` | 红色组件边缘硬度，贴图边界通常更硬 |
| `red_component_color_std` | 红章内部颜色标准差，贴图颜色可能过平 |

当前风险原因：

- `red_stamp_like_region`
- `seal_red_connected_component`
- `seal_hard_edge_overlay`
- `seal_flat_color_overlay`
- `pdf_smask_present`

当前评分策略：

- 红色区域、红色连通组件、颜色过平只作为弱证据。
- `seal_hard_edge_overlay`、`pdf_smask_present`、密集覆盖层同时出现时才构成更强证据。
- 扫描件来源会降权，避免真实红章或扫描边缘被误报。

仍需增强：

1. 先用目标检测模型定位印章区域。
2. 对印章区域提取颜色、边缘、纹理、噪声特征。
3. 与文档背景区域做一致性比较。
4. 建立印章模板库，检测重复使用和异常贴图。
5. 增加圆形/椭圆形形态、文字环形分布、透明边 alpha 过渡检测。

### 4.4 字体与文本层异常

当前离线通过 Poppler 字体警告做粗检测。

可检测特征：

- 同一段文字字体不一致。
- 数字区域字体与周围文字不一致。
- 字符间距、基线、字号异常。
- PDF 文本层缺失，但视觉上有文字。
- 文本层内容与渲染图 OCR 内容不一致。

未来方案：

1. 提取 PDF 文本层。
2. 渲染页面图像。
3. OCR 识别页面视觉文本。
4. 比较文本层与 OCR 结果。
5. 对金额、日期、姓名等敏感字段做局部一致性检测。

### 4.5 业务逻辑一致性检测

当前已实现基础版，入口为 `src/analyze_text_business_rules.py`，并已接入综合评分。

当前可检测内容：

| 材料 | 逻辑规则 |
| --- | --- |
| 发票 | 价税合计 = 金额 + 税额 |
| 银行流水 | 账号缺失、金额序列过稀；余额递推待增强 |
| 合同 | 甲乙方缺失、日期缺失；大小写金额一致待增强 |
| 征信报告 | 身份证号缺失、日期缺失；账户统计一致性待增强 |
| 回单/结算单 | 金额、账号核心字段缺失；合计一致性待增强 |

实现方式：

1. OCR 或 PDF 文本抽取。
2. 字段定位和结构化解析。
3. 字段格式校验。
4. 跨字段逻辑校验。
5. 输出不一致证据。

## 7. 结论

当前系统已经能做 PDF 结构风险初筛，但虚假样本数量不足，风险规则也偏基础。

下一阶段的关键不是单纯调模型，而是：

1. 按材料类型补充多样化 fake 样本。
2. 给每个 fake 样本记录伪造方式。
3. 建立人工标注和复核闭环。
4. 扩展图片 PS、印章贴图、OCR 业务逻辑检测。
5. 用更均衡的数据重新评估准确率、召回率和误报率。

### 图片/页面视觉取证检测

脚本：

```text

```

支持输入：

- PDF 首页渲染图
- JPG/JPEG
- PNG

检测字段：

| `block_variance_score` | 局部噪声块一致性，检测局部编辑区域 |
| `edge_density` | 边缘密度，检测贴图边界或异常锐化 |
| `red_stamp_score` | 红色区域比例，辅助识别印章/红章贴图 |
| `visual_risk_score` | 视觉取证规则分 |
| `visual_risk_reasons` | 视觉风险原因 |

当前已生成：

```text

```

本轮采用抽样策略：

- 全部 fake 样本
- 全部图片样本
- 每类 normal 抽样若干份

原因是全量 PDF 渲染和图像取证计算较慢，后续可异步跑全量。

### 当前检测效果观察

增强 PDF 对象规则后，PDF 特征表统计为：

- 对扫描图、截图图、红章贴图类更有意义。
- 对 reportlab 生成的干净矢量 PDF 伪造样本不敏感。
- 这说明合成样本还需要增加“真实扫描底图 + PS 局部篡改 + 再转 PDF”的类型。
