## 当前问题

- 可视化仍偏数据看板，缺少围绕“单份材料核查”的完整工作流。
- 原 PDF、检测证据、模型判断和人工结论没有形成闭环。
- 线上服务不直接依赖 GPU，重型图像模型训练仍需单独训练环境或容器化。
- 当前规则评分已能覆盖基础伪造样本，但正常扫描件误报仍需持续校准。
- 需要减少对显式训练水印的依赖，补充更真实的 hard negative。

## P0：立即提升体验

- 支持原 PDF 在线预览：
  - 样本详情抽屉内嵌 PDF。
  - 支持 HTTP Range，浏览器可按需加载大 PDF。
  - 样本表显示“可预览/未同步”状态。
- 支持上传 PDF 检测：
  - 上传文件保存到 `data/uploads`。
  - 后端纯 Python 扫描 PDF 对象结构。
  - 输出页数、图像对象数、SMask、Creator、Producer、风险原因和风险分。
  - 上传样本进入样本核查列表，可再次预览。
- 支持上传样本人工标注：
  - 上传后默认标记为 `uploaded`，不直接进入训练。
  - 人工标注为 `normal/fake` 后进入下一轮对象特征模型训练。
  - 可撤回标注，避免错误样本污染训练集。
- 支持交互式核查：
  - 搜索、筛选、排序。
  - 点击样本进入详情。
  - 图表点击联动样本表。
